# what is clean code

-   [참고 자료(Refactoring.Guru)](https://refactoring.guru/ko/refactoring/technical-debt)

1. 다른 개발자에게 분명한 코드
2. 중복이 포함되지 않는 코드
3. 최소한의 클래스와 동작을 감지하는 부분들이 있는 코드
4. 대부분의 테스트를 통과하는 코드
5. 유지관리가 쉽고 비용이 적게 드는 코드

## 기술 부채

새로운 기능에 대한 테스트를 작성하지 않고도 일시적으로 속도를 높일 수 있지만, 결국 테스트를 작성하여 빚을 갚을 때까지 매일 진행 속도가 점차 느려지는 것.

<br />

### 기술 부채의 원인

1. 비즈니스 압력
2. 기술 부채의 결과에 대한 이해 부족
3. 구성 요소의 엄격한 일관성을 유지하지 못함
4. 테스트 부족
5. 문서 부족
6. 팀원 간 상호작용 부족
7. 여러 지점에서 장기간 동시 개발
8. 지연된 리팩토링
9. 규정 준수 모니터링 부족
10. 무능력
